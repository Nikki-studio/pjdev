
# editor rules
- datatypes eg. `void` `char` `int` `float` `double` get color **blue** (`COLOR_BLUE`)
- function definition and their fields and brackets get the same color **yellow** (`COLOR_YELLOW`)
- function call and its brackets get color **magenta** (`COLOR_MAGENTA`) [closest to gold accent]
- user defined data structures get color **cyan** (`COLOR_CYAN`) [blue-gray replacement]
- operator symbols get color **red** (`COLOR_RED`) eg `+` `++` `-` `--` `*` `/` `^` `!` `&` `|`
- variables get color **green** (`COLOR_GREEN`) [turquoise replacement]
- variables in containers eg `array`, `struct`, `vector`, e.t.c. get a variation of **cyan** (`COLOR_CYAN`) [light turquoise replacement]
- all numbers eg `1234567890` or `1234567890.87654321` get color **white** (`COLOR_WHITE`) [brown replacement]
- chars `'*'` and strings variations `"**"`, ``<backtick>**<backtick>``, `"""***"""`, `'''***'''` get color **green** (`COLOR_GREEN`)
- logical symbols eg. `||` `&&` `!=` `==` `>` `<` `>=` `<=` ... get the color **magenta** (`COLOR_MAGENTA`) [pink replacement]
- brackets following a variable name or function name meant to manipulate through the functions get the color of the subject variable or function
- endings eg `;` of placeformaters eg. `%s` `%i` ... get the color **white** (`COLOR_WHITE`)
